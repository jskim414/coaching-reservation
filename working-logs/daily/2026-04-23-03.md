# 작업 로그

- 작업일: 2026-04-23
- 순번: 03
- 범위: 공개 예약 화면 정리 + 운영 문자/설정 보강 + 작업로그 정리 + Vercel 보안 점검

## 작업 목표

- 공개 예약 화면을 실제 고객이 바로 사용할 수 있는 수준으로 정리
- 입금 계좌와 문자 문구를 운영자가 관리할 수 있게 보강
- 작업로그 폴더 구조를 정리
- Vercel 배포 전 프런트 민감정보 노출 가능성을 점검

## 구현 내용

### 1. 공개 예약 흐름 정리

- 예약 신청 화면 문구를 간결하게 정리
- 예약 조회 흐름을 이름 + 전화번호 + 비밀번호 기준으로 정리
- 예약 신청 필수항목을 이름, 이메일, 전화번호, 환불 계좌 정보, 예약 조회 비밀번호로 고정
- 메인 화면에 사전 신청 안내 문구와 Google Form 링크를 연결

### 2. 관리자/운영 기능 보강

- 관리자 예약 삭제, 일괄 삭제, 기간 필터 추가
- 관리자 운영 안내 설정에서 입금 계좌를 수정할 수 있도록 추가
- 예약 신청 문자에 입금 계좌와 12시간 입금 기한 반영
- 예약 확정 문자에 신청 이메일을 직접 포함하도록 수정
- 관리자 경로를 `/admin8630`으로 변경

### 3. 작업로그 폴더 리팩토링

- `working-logs/daily/` 폴더에 날짜별 작업 로그 이동
- `working-logs/orders/` 폴더에 요청 원문 이동
- 루트에는 `work-log.md`, `to-do-list.md`, `README.md`만 유지

### 4. Vercel 보안 점검 및 조치

- `client/vite.config.js`의 환경변수 노출 범위를 `VITE_*`로 제한
- Google Client ID만 명시적으로 프런트에 주입하도록 수정
- 프런트 코드에서 `import.meta.env.GOOGLE_CLIENT_ID` 직접 참조를 제거
- 키워드 검색 기준으로 SOLAPI, Telegram, 세션 시크릿이 프런트 번들에 직접 포함되는 흔적은 확인하지 못함

## 점검 결과

- 프런트에서 가장 큰 노출 위험은 `envPrefix: ["VITE_", "GOOGLE_"]` 설정이었다.
- 해당 설정은 `GOOGLE_`로 시작하는 모든 환경변수를 프런트 빌드에 노출할 수 있어, 향후 `GOOGLE_CLIENT_SECRET` 같은 값이 추가되면 그대로 새어 나갈 수 있었다.
- 현재 구조에서는 Google Client ID만 프런트에 노출되고, 서버용 키는 서버 코드에만 남도록 조정했다.
- 다만 Vercel에서는 SQLite 파일과 `operation-settings.json` 파일 쓰기 방식이 지속되지 않으므로, 현 구조 그대로는 운영 배포에 부적합하다.
