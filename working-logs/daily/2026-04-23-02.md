# 작업 로그

- 작업일: 2026-04-23
- 순번: 02
- 범위: Google 로그인 검증 안정화

## 작업 목표

- `.env`에 이미 등록된 Google 설정을 기준으로 관리자 로그인 성공 가능성을 높이기
- Google Identity Services localhost 개발 조건에 맞는 프런트 메타 설정 보강

## 구현 내용

### 1. Google ID token 검증 방식 교체

- 기존 수동 JWK 검증 로직 제거
- `google-auth-library` 기반 `verifyIdToken()` 방식으로 교체

적용 이유:

- Google 공식 Node 라이브러리 기준으로 검증하도록 맞춤
- `aud`, `iss`, `exp`, 서명 검증을 라이브러리 레벨에서 처리
- 실환경에서 토큰 검증 실패 가능성을 줄이기 위함

### 2. 프런트 메타 설정 보강

- `client/index.html`에 `Referrer-Policy` 메타 추가
- `Cross-Origin-Opener-Policy: same-origin-allow-popups` 메타 추가

적용 이유:

- localhost 개발 중 Google 로그인 버튼/팝업 호환성 보강
- Google Identity Services 가이드 기준 반영

## 확인 사항

- `.env`에서 `GOOGLE_CLIENT_ID` 로드 확인
- 프런트에서 `GOOGLE_CLIENT_ID` 노출 설정 확인
- `admins` 테이블에 허용 관리자 이메일 존재 확인

## 검증 내용

- `npm run client:build` 통과
- 인증 관련 모듈 로딩 확인

- `package.json`
