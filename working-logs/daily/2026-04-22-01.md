# 작업 로그

- 작업일: 2026-04-22
- 순번: 01
- 범위: 관리자 운영 기능 확장 + 테스트 환경 SMS 비활성화

## 작업 목표

- 관리자 화면에서 코칭/워크샵 프로그램을 생성, 수정, 삭제할 수 있도록 구현
- 관리자 화면에서 프로그램별 일정을 생성, 수정, 삭제할 수 있도록 구현
- 프로그램별 예약 현황을 조회하고 예약 상태를 변경할 수 있도록 정리
- 최종 제품 완성 전까지 테스트 과정에서는 SOLAPI SMS가 실제 발송되지 않도록 조치

## 구현 내용

### 1. 관리자 서비스 CRUD API 추가

- `GET /api/admin/services`
- `POST /api/admin/services`
- `PATCH /api/admin/services/:serviceId`
- `DELETE /api/admin/services/:serviceId`

적용 내용:

- 코칭/워크샵 타입 관리
- 프로그램명, 설명, 진행 시간, 가격, 기본 정원, 활성 여부 관리
- 예약 이력이 있는 서비스는 삭제되지 않도록 보호
- 먼저 연결된 슬롯을 정리해야 삭제 가능하도록 제약 추가

### 2. 관리자 슬롯 CRUD API 추가

- `GET /api/admin/slots`
- `POST /api/admin/slots`
- `PATCH /api/admin/slots/:slotId`
- `DELETE /api/admin/slots/:slotId`

적용 내용:

- 서비스별 일정 조회 지원
- 시작 시각, 종료 시각, 정원, 오픈 여부 관리
- 예약 이력이 있는 슬롯은 삭제되지 않도록 보호
- 예약 이력이 있는 슬롯은 다른 서비스로 이동되지 않도록 보호
- 확정 인원보다 작은 정원으로 수정하지 못하도록 제약 추가

### 3. 관리자 프런트 운영 화면 확장

관리자 화면에서 다음 기능을 사용할 수 있도록 구현:

- 프로그램별 예약 조회
- 예약 상태 필터
- 예약 상세 조회
- 예약 상태 변경
- 프로그램 생성 / 수정 / 삭제
- 프로그램별 일정 생성 / 수정 / 삭제

추가한 UI:

- 프로그램 관리 패널
- 일정 관리 패널
- 관리자 운영 데이터 새로고침 버튼
- 선택된 프로그램 기준 일정 목록
- 일정/프로그램 수정용 폼

### 4. 테스트 환경 SMS 비활성화

환경변수 추가:

- `SMS_ENABLED=false`

적용 방식:

- `SMS_ENABLED`가 `true`가 아니면 SOLAPI 발송을 수행하지 않음
- 테스트 중에도 메시지 로그는 남기되 상태를 `skipped`로 기록
- 최종 오픈 전 `.env`에서 `SMS_ENABLED=true`로 바꾸면 재활성화 가능

## 검증 내용

### API 검증

- 관리자 서비스 목록 조회 확인
- 관리자 슬롯 목록 조회 확인
- 관리자 서비스 생성/수정/삭제 확인
- 관리자 슬롯 생성/수정/삭제 확인
- 예약 상태 `confirmed` 변경 후 다시 `requested`로 원복 확인

### SMS 비활성화 검증

- 테스트 예약 생성 시 `message_logs`에 SMS가 `skipped`로 기록되는 것 확인
- 예약 확정 시 확정 SMS도 `skipped`로 기록되는 것 확인
- Telegram 알림은 기존대로 동작 유지

### 프런트 검증

- `npm run client:build` 통과
