# 작업 로그

- 작업일: 2026-04-23
- 순번: 01
- 범위: 인증 정식화 + 예약 메모/메시지 로그 + 문서 정리

## 작업 목표

- 관리자 인증을 임시 헤더 방식에서 Google 로그인 + 세션 방식으로 전환
- 예약 메모 API와 UI를 추가
- 메시지 로그 조회 API와 UI를 추가
- README와 작업 로그를 현재 상태에 맞게 갱신

## 구현 내용

### 1. 관리자 인증 정식화

- `POST /api/auth/google`
- `POST /api/auth/logout`
- `GET /api/auth/me`

적용 내용:

- Google ID token 검증 로직 추가
- 허용된 관리자 이메일만 로그인 가능하도록 유지
- HttpOnly 쿠키 기반 세션 적용
- 기존 `x-admin-email` 임시 인증 제거

### 2. 예약 메모 / 메시지 로그

- `PATCH /api/admin/bookings/:bookingId/memo`
- `GET /api/admin/bookings/:bookingId/message-logs`

적용 내용:

- 관리자 화면에서 내부 메모 저장 가능
- 예약 상세 화면에서 메시지 로그 확인 가능
- DB 스키마 변경 없이 기존 `bookings.note` 컬럼을 직렬화해 예약자 메모와 관리자 메모를 함께 저장

### 3. 관리자 화면 개선

- Google 로그인 화면 추가
- 세션 만료 시 관리자 로그인 화면으로 복귀
- 예약 상세에서 내부 메모 저장
- 예약 상세에서 메시지 로그 조회
- 서비스/슬롯 관리 화면은 쿠키 세션 기반으로 동작하도록 수정

### 4. 에러 응답 정리

- `ok: false`
- `message`
- `error.code`
- `error.message`
- `error.details`

## 검증 내용

### 빌드 검증

- `npm run client:build` 통과

### 서버/API 검증

- `GET /health` 정상 응답 확인
- `GET /api/services` 정상 응답 확인
- `GET /api/auth/me` 비인증 401 확인
- `GET /api/admin/bookings` 비인증 401 확인
- `POST /api/bookings` 성공 1건 확인
- `POST /api/bookings` 필수값 누락 400 확인
- `POST /api/auth/google` 잘못된 토큰 400 확인
